## Bugs Found (Not Fixed)

### 1. CurrentSectionIndex pre-plan value
- **Location:** ISessionTimerService.cs doc vs SessionTimerService.cs
- **Issue:** Doc says `-1` when no plan; actual is `0` (C# default, never set to `-1`)
- **Handling:** Tests assert actual value; doc discrepancy noted in comments

### 2. ComputeBehindSchedule double-clock-read
- **Location:** SessionTimerService.cs — ComputeBehindSchedule()
- **Issue:** Calls `ComputeSessionElapsed()` twice (reading `_clock.Elapsed` at T1 and T2 > T1); phantom lag ~100–300 ns produces spurious positive BehindSchedule even when on-schedule
- **Handling:** Test tolerates with `< 10ms` rather than `== 0`; noted for polish phase
